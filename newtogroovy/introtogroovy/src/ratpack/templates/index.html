What's Groovy

  • Dynamic, compiled language for the Java Virtual Machine(JVM).
  • Interoperable with existing Java libraries
  • Concise, easy to read code
  • Used by many major companies including:
Groovy is the main focus of today’s session and the foundation of the Gr8 ecosystem of technologies. To start, Groovy is a dynamic compiled language for the JVM. That mean’s it’s dynamic like languages you may know like Python and PHP, but compiled for efficiency to run on the Java Virtual Machine aka the JVM. Many of you may be asking, why learn yet another programming language? And granted, Groovy is not the best language for every situation. However, in a setting where productivity is paramount and you are looking for the stability of the known architecture stack like Java, Groovy is the best choice. As you’ll see, Groovy’s easy to learn, flexible syntax and interoperability with existing Java libraries, make it a great language to add to your repertoire. It's not just a prototyping solution and many companies use Groovy for enterprise solutions including Netflix, LinkedIn, and Mutual of Omaha.
  • Open source -- maintained by Pivotal and programmers like you!
  • http://groovy-lang.org
  • contribute: https://github.com/groovy/groovy-core
Groovy is also open source. The repository for core is on github and anyone can submit a PR. It's under active development and the community is pretty friendly and welcoming to newcomers.

You can do anything you can do in Java in Groovy. In fact, Almost all Java code is valid Groovy code. Groovy, however, usually has a way to do things with less typing than Java. One big thing you’ll notice is that in Groovy, you don’t always have to declare a function and then call it. Since Groovy is a scripting language, you can just put commands outside of any function scope and it is valid Groovy. Semicolons are unnecessary.

  • Strings
  • Collections
    • Lists

We will look at a few datatypes in this workshop: In addition to your normal primitives(int, bool, etc), there are Strings, Collections, and Closures. Collections include classes such as Lists and Maps. Groovy Lists correspond with Java ArrayLists and Groovy Maps correspond with Java HashMaps. Closures are pieces of functionality that you can pass around as data. Groovy is optionally typed. When we don’t care what type a variable is or we want mixed typing, we can use def instead of a data type. Def, short for ‘define’, roughly translates to Java Object, the base class for all other classes.

Strings

(examples in console)

  • Single Quoted
  • Double Quoted
  • Multi-line
Groovy has many types of strings, but today we’ll go over the three main types: Single Quoted, Double Quoted and Multi-Line Strings. First the single quoted string is for plain text. For example, ‘Hello, World!’ Double Quotes can be used to inject code into the text of the string. For example, “Hello, (dollar) name”. Name is a variable, the dollar sign tells Groovy to inject its value into the string. If you need a particularly long section of text, use multi-line strings by denoting three quotes followed by your text and ended by another three quotation marks. Just like before, use single quotes for plain text and double quotes if you need to embed code. You can manipulate Strings easily using the automatically generated String function split and the List function join. Pass a string to tokenize and it will split the String based on that as a delimiter. I heart Groovy is split on the spaces and the heart to create two Strings, I and Groovy. Pass a token to join and a list of Strings will be transformed with that token inserted as a separator. I, heart, and Groovy can be transformed to the single String, I heart Groovy.

The first and simplest type of Groovy collection is a list. Lists can have elements of different types– the same List can hold both Strings and integers. To define a list, bound your list in square brackets and separate the items with commas. To access elements of the list, use square brackets. Just like in Java, indices start at 0, so the index of the number 2 is 2, not 3. Here, we have changed the third element in the list to 3. You can easily append elements onto the end of the list using the shovel operator, which is two “less than” signs. To append means to add to the end. Collections have many available methods including basic mathematic operations plus, minus, and multiply. Here, I can use my list and multiply it by two. When you multiply the entire collection, it takes all of the elements in the list and duplicates them.

Maps

(examples in console)

  • definition
  • get/set by keys
  • get all keys/values
Maps are like lists but with keys instead of numerical indices. The keys and values are separated by colons then separated from other pairs with commas. You can access the values in a Map using a dot then the keyname. This is like getting a member of a Java class. For example, we can do myMap dot val1 to get the String a. This works for both getters and setters. To set the value for the key val4 to 5, we can use myMap.val4 = 5. You can also separate out the keys and values by accessing the .values() and .keySet() methods on the map. Here we see the values of myMap displayed as a list.

Closures

Code as Data

(to the console!)

  • code as data
  • definition
  • call
Now, we’ll talk about closures. Closures are not completely unique to Groovy and other languages use similar concepts. Recently, Java 8, introduced a similar feature -- lambdas. By definition, closures are code as data. That means that they are pieces of functionality that can be passed around like variables including as a parameter to another function. Here is the syntax for defining and using a closure. It is like defining a variable; we start with specifying a type and a name. After the equals sign we include the body of the function in curly braces. Unlike a standard function definition, parameters are included immediately after the first curly brace and before an arrow which indicates the start of the body. Multiple parameters are separated by commas and any default parameters are indicated with an equals followed by the default value. You can call a closure by using the built-in function called “call” followed by the parameter(s). myClosure has two parameters, named name and year. You could have also have only one or no parameters at all. Typing your parameters is also optional. Just like with functions, return statements are optional. Unless otherwise specified, the last line of a closure is what's returned.

False values

  • Null

Before evaluating functions and other paradigms, it's important to note how Groovy handles logic. Unlike other languages, truth is very loosely defined. Unless otherwise specified, the default values for are to evaluate null, zero, empty set, and empty strings as false. It's important to note the zero here when working with number values.

Basic Loops


        List dl = [];
        for(int i=1; i<4; i++){
            dl.add(i * 2);
        }
        println dl;
        --> [2,4,6]
    

        List dl = []
        for(i in [1,2,3]) {
        dl << i * 2
        }
        println dl
        --> [2,4,6]
Basic Groovy for loops are very similar to Java for loops, but in Groovy, you can very easily iterate over a list of things. In Java, you’re restricted to iterating over a range of numbers. So, you would have to iterate over the indices and then index into the list of numbers that you want to double. In the Groovy loop, we declare the variable i to be our iterator. First we grab 2 from the list and set I equal to it. Then we double it. Then we grab 4 and set I equal to it, then 6. We could have done this same loop many ways in Groovy.

Each

Map m = [val1: 2, val2: 4, val3: 6]
        List dl = []
        m.each { key, value ->
            dl << value * 2
        }
        println dl
        --> [4, 8, 12]
You can iterate over every item of a List or Map using the method each. Each is a function that takes a closure as a parameter and applies that closure to every item in the collection. Here we have a Map with three key-value pairs. In our each loop we separate the key and value for each pair and then add only the new doubled value to our result list. You’ll see it has the same result as our previous example. You can see the each function’s equivalence as a loop. Each is basically a function that performs the closure as the body of a for loop. Using each, we can avoid the boilerplate “for in” syntax.

Collect


        Map m = [var1: 2, var2: 4, var3: 6]
        List dl = m.collect { name, value ->
            value * 2
        }
        println dl
        --> [4, 8, 12]
In cases where we are aggregating results, we may want to use a collect statement instead of an each. The effect is the same except that it collects all of your results for you rather than having to automatically append the answer into your data structure. Here we don’t have to initialize a new list for our answer myDoubledList. We just set it equal to the result of the .collect call. Then, remembering that by default closures return the last executed statement, we can just leave it as value * 2 and the results of doubling each item from our myList is saved in myDoubledList.

Spread Dot


        (1..3).multiply(2)
        -->[1,2,3,1,2,3]
        (1..3)*.multiply(2)
        --> [2,4,6]
If you are purely dealing with lists and single functions, Groovy has another unique function that makes code even shorter. That function is Spread Dot. Continuing with our myList example, we are going to add a spread dot at the end of the variable name. This function takes the collection and applies the multiply function to each element of the collection individually rather than the collection as a whole. This means we get similar results to our each and collect loops, but completely different results than when we multiplied the list as a whole times two.

Search

Map m = [val1: 2, val2: 4, val3: 6]

find

m.find{it.value % 2 == 0 && it.value % 3 != 0}
    --> val1:2

findAll

m.findAll{it.value % 2 == 0 && it.value % 3 != 0}
    --> [val1:2,val2:4]
Find returns the first value in a collection for which the closure evaluates to true. FindAll returns a collection of all matching values. It is the name of the default closure parameter. If you don’t don’t declare any parameters, the item in the collection will be denoted by the variable named it. In these examples, we look for any element in the list which is divisible by 2 but not by 3. Find returns only the first key/value pair, and FindAll returns both pairs that fit this description.
